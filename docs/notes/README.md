# Notes Business Logic: MoSCoW Plan (as of 2026-03-26)

## Must Have (core requirements for notes management)
- [x] Create, read, update, and delete notes (CRUD)
- [x] Each note is associated with a user (ownership enforced)
- [x] Notes have title, content, and timestamps (created/modified)
- [ ] Only authenticated users can access their own notes
- [ ] Note sharing with other users (view or edit permissions)
- [ ] Attachments (images, files)

## Should Have (improvements for usability and features)
- [ ] Search and filter notes by title/content
- [ ] Tagging or categorization of notes
- [ ] Soft delete (trash bin/restore)
- [ ] Pagination and sorting for note lists

## Could Have / Won't Have (for now)
- [ ] Note version history
- [ ] Rich text formatting

---
This MoSCoW plan outlines the current and planned business logic for the notes domain. "Must Have" items are required for a functional baseline. "Should Have" and "Could Have" are recommended for enhanced usability and features. "Won't Have" are explicitly out of scope for now.
